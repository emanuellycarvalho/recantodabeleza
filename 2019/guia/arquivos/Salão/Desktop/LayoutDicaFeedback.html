Categoria: Diretrizes Gerais de Layout

Orientação: Feedback

  • Mantenha padrões de feedback por todo o sistema;
  • Informe o usuário de como foi interpretado o envio de seu formulário:
    • Faça-o por meio de mensagens e do realce do campo em questão;
    • Escreva mensagens curtas e claras, para que haja agilidade ao encontrar a solução;
  • Defina o ponteiro do mouse em forma de mão para áreas clicáveis, como botões e tabelas.
  • Por exemplo:
    • Campos ainda não editados ou preenchidos corretamente estão em cinza;
    • Foi utilizado azul para indicar o campo atualmente selecionado (em edição);
    • O vermelho foi escolhido para mostrar campos preenchidos incorretamente.
  • Monte suas mensagens de preenchimento obrigatório ou errado conforme um padrão:
    • Neste projeto exemplo, campos com máscara, como CPF recebem um alerta do tipo: "Digite um CPF válido";
    • Campos sem máscara, por sua vez, têm alertas com o conteúdo: "Preencha este campo".
  • Escolha um padrão para quando acontecerá o realce dos campos preenchidos, no nosso sistema:
    • Os campos dos formulários de cadastro informam ao usuário que foram preenchidos de forma errada imediatamente.
      • Dado que esses formulários costumam ser preenchidos sempre de cima para baixo;
  • Outas telas, como a de agendamento, só realçam os campos errados quando o usuário resolve salvar o registro.
    • Uma vez que o preenchimento costuma ser feito sem uma sequência definida.
