Diretrizes da Tela Principal

  • Navegação: todo Sistema Web deve ser navegável, isto é, fornecer maneiras de ajudar os usuários a navegar, localizar conteúdos e determinar onde se encontram. Em Sistemas Web, o uso de mecanismos de navegação adequado é fundamental para que o usuário não se perca no sistema.
    • A seguir lista-se alguns recursos para uma navegação adequada:(1)
    • É comum que em todo Sistema Web, ao se clicar na logo ou no título, o usuário é encaminhado à Página Inicial;
    • Neste projeto exemplo, quando entra-se no sistema e clica-se em Minha Conta, o sistema redireciona o usuário para o login. Após logado, se o usuário for a secretária, ao se clicar em Minha Conta, o usuário é redirecionado para a página principal da secretária e se o usuário for cliente, é redirecionado para a página inicial de cliente;
    • Deve-se bloquear um usuário não logado acessar outras do sistema pela URL. Deve-se, também, bloquear que um cliente acesse páginas permitidas apenas à secretária.
    • Atenção: neste projeto exemplo não foi utilizado uma ferramenta bastante importante à navegação: detalhar o caminho que o usuário já percorreu e permitir que ele volte às páginas anteriores à que ele se encontra.
      • Exemplo: Guia de interface -> Interface Web -> Tela Principal
      • Isso é importante para que, em tarefas que exigem várias etapas para serem concluídas, o usuário não se perca.
  • Barra de Menus: esse tipo de menu é usado para exibir um conjunto de menus em uma linha horizontal, normalmente na parte superior de uma janela. Todas as regras do Menu Pull-down se aplicam a ele também:(2)
    • Os títulos de menu representam todo o menu e devem, portanto, comunicar tão claro quanto possível o propósito de todos as opções;
      • Sempre que possível, use apenas uma palavra, não composta e curta para os títulos de menu;
      • Utilize, na inicial do título, letra maiúscula;
    • Quanto ao número de opções nos Menus Pull-down, restrinja a, no máximo, 8 (oito) itens;
    • Organize as funções em categorias segundo uma lógica, por exemplo, os CRUDs em uma categoria, as funções de relatório em outra.
  • Cockpit: aproveite o espaço disponível na Tela Principal para mostrar informações úteis, como lembretes e informações urgentes:(2)
    • Neste caso, foram escolhidas tanto a exibição de Serviços quanto de Produtos para ficar no Cockpit.(2)
