Diretrizes de Relatório

  • Permita ao usuário procurar objetos que podem ser facilmente confundidos, como Cliente, por meio de um Campo de Seleção, que traz mais de uma informação, sendo que uma delas necessariamente deve ser exclusiva de um cliente, como, no projeto exemplo, o número de telefone;
  • Verifique quais filtros devem ser obrigatórios e quais não precisam ser:
    • Cliente claramente será exigido, dado que todos os itens da lista dependem de qual cliente for selecionado;
    • No caso das datas, o desenvolvedor deve avaliar a necessidade de obrigar ou não o preechimento delas. Porém, note que permitir que nenhuma data seja preenchida pode causar lentidão ao sistema se houver muitos itens registrados.
  • Prefira, no caso de datas, dar ao usuário a opção de preencher a data completa de início e fim:
    • Isso porque liberá-lo apenas a opções como Semanal ou Mensal é demasiado limitante.
  • Coloque as colunas da tabela iguais as colunas mostradas no relatório;
  • Inclua, no caso de objetos com informações adicionais, um botão para acessar essas informações em cada linha da tabela, como o botão com ícone de olho;
  • Insira um botão para retornar à tela anterior além do usado pelo fechar do sistema operacional. Neste caso, este é o botão Cancelar.
  • Habilite o botão Exibir apenas quando todos os campos obrigatórios forem preenchidos;
